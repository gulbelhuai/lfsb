# 预到龄发放通知测试准备清单

## 目标

本文档只覆盖测试前准备，不包含实际执行测试。

## 一、主测试环境准备

- 确认后端地址：`http://localhost:8087/api`
- 确认前端地址：`http://localhost:81`
- 确认 `.env` 中数据库、Redis、文件目录配置正确
- 确认数据库已执行 `sql/refactor/13_benefit_notice_batch_flow.sql`
- 确认 `PROFILE_PATH`、`LOG_PATH` 可写

## 二、依赖准备

前端依赖：

```powershell
cd ruoyi-ui
npm install
npx playwright install
```

脚本依赖：

```powershell
cd sql
npm install
```

## 四、角色与账号准备

建议至少准备以下账号：

- 经办人：`operator01`
- 复核人：`reviewer01`

如需真实 UI 自动化，建议通过环境变量显式配置：

```powershell
$env:PLAYWRIGHT_OPERATOR_USER='operator01'
$env:PLAYWRIGHT_OPERATOR_PASSWORD='Test@12345'
$env:PLAYWRIGHT_REVIEWER_USER='reviewer01'
$env:PLAYWRIGHT_REVIEWER_PASSWORD='Test@12345'
```

## 五、真实 UI 自动化预备配置

默认脚手架不会自动执行真实链路，需要显式开启：

```powershell
cd ruoyi-ui
$env:PLAYWRIGHT_BASE_URL='http://127.0.0.1:81'
$env:RUN_REAL_E2E='true'
```

可选测试月份：

```powershell
$env:PLAYWRIGHT_NOTICE_MONTH='2030-02'
```

## 五点一、Cursor 内置浏览器补证准备

- 确认 Cursor 内置浏览器可打开前端地址 `http://127.0.0.1:81`
- 优先复用当前已登录页签，避免重复登录影响验证效率
- 若 Playwright 在登录、跳转、弹窗、上传等步骤不稳定，可立即改用 Cursor 内置浏览器补证
- 补证时建议至少记录：
  - 登录账号
  - 实际访问 URL
  - 关键批次号/记录号
  - 看到的页面结果

## 六、测试前检查命令

仅生成准备摘要，不执行测试：

```powershell
node .\tests\准备预到龄发放通知测试环境.js
```

该脚本会输出：

- 环境摘要
- 关键表检查结果
- 关键账号检查结果
- 测试数据文件是否存在
- 报告输出目录是否已创建

## 七、后续执行入口

注意：以下命令仅作为后续执行入口预留，本次准备阶段不执行。
