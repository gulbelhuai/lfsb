# 本地启动后台服务说明

## 前置条件

### 1. 环境要求
- **JDK**: Java 17 或更高版本
- **MySQL**: 8.0 或更高版本
- **Redis**: 3.0 或更高版本（可选，但建议安装）

### 2. 数据库准备
1. 创建数据库（如果不存在）：
   ```sql
   CREATE DATABASE langfang_shebao CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
   ```

2. 导入数据库脚本：
   - 执行 `sql/langfang-shebao.sql` 文件
   - 执行 `sql/quartz.sql` 文件（定时任务相关）

3. 检查数据库配置（`ruoyi-admin/src/main/resources/application-druid.yml`）：
   - 默认配置：
     - 主机: `localhost`
     - 端口: `3306`
     - 数据库: `langfang_shebao`
     - 用户名: `root`
     - 密码: `root`
   - 如果配置不同，可以通过环境变量修改：
     - `MYSQL_HOST`: 数据库主机
     - `MYSQL_PORT`: 数据库端口
     - `MYSQL_DB`: 数据库名
     - `MYSQL_USER`: 用户名
     - `MYSQL_PWD`: 密码

### 3. Redis准备（可选但建议）
- 默认配置：
  - 主机: `localhost`
  - 端口: `6379`
  - 数据库索引: `5`
  - 密码: 无
- 如果配置不同，可以通过环境变量修改：
  - `REDIS_HOST`: Redis主机
  - `REDIS_PWD`: Redis密码

## 启动方式

### 方式一：使用启动脚本（推荐）

1. **Windows系统**：
   ```bash
   # 双击运行或在命令行执行
   start.bat
   ```

2. **Linux/Mac系统**：
   ```bash
   # 需要先创建启动脚本或使用方式二
   ```

### 方式二：直接运行jar文件

1. 打开命令行，进入项目目录
2. 执行以下命令：
   ```bash
   java -Xms512m -Xmx1024m -XX:MetaspaceSize=128m -XX:MaxMetaspaceSize=512m -jar ruoyi-admin/target/shebao-server-20260118-1500.jar
   ```

### 方式三：在IDE中运行

1. **IntelliJ IDEA**：
   - 打开项目
   - 找到 `ruoyi-admin/src/main/java/com/ruoyi/RuoYiApplication.java`
   - 右键选择 "Run 'RuoYiApplication.main()'"

2. **Eclipse**：
   - 导入项目
   - 找到 `RuoYiApplication.java`
   - 右键选择 "Run As" -> "Java Application"

### 方式四：使用Maven命令

```bash
# 在项目根目录执行
cd ruoyi-admin
mvn spring-boot:run
```

## 常见问题

### 1. 端口被占用
如果8087端口被占用，可以修改 `ruoyi-admin/src/main/resources/application.yml` 中的端口配置：
```yaml
server:
  port: 8088  # 修改为其他端口
```

### 2. 数据库连接失败
- 检查MySQL服务是否启动
- 检查数据库配置是否正确
- 检查数据库用户权限

### 3. Redis连接失败
- 如果未安装Redis，可以暂时忽略（某些功能可能不可用）
- 或者安装Redis并启动服务

### 4. 内存不足
如果启动时出现内存不足错误，可以调整JVM参数：
```bash
java -Xms256m -Xmx512m -jar ruoyi-admin/target/shebao-server-20260118-1500.jar
```

## 停止服务

- **Windows**: 在运行窗口按 `Ctrl+C`
- **Linux/Mac**: 在终端按 `Ctrl+C`
- 或者找到Java进程并kill：
  ```bash
  # Windows
  taskkill /F /IM java.exe
  
  # Linux/Mac
  ps aux | grep java
  kill -9 <PID>
  ```
