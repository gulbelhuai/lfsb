# 失地居民信息修改对其他补贴身份的影响分析

## 📅 修改时间
2025-10-18

## 📋 修改概述
本次主要修改失地居民信息管理功能，涉及基础人员信息表（`shebao_subsidy_person`）的重大调整。由于四种补贴身份（失地居民、拆迁居民、被征地参保补贴、村干部）都共享同一个基础人员信息表，因此本次修改会对其他三种补贴身份产生直接影响。

### 1.2 用户编号生成规则

**格式**：`AAA-BBB-CCCC`

- AAA：街道办编码（3位数字，001-999）
- BBB：村委会编码（3位数字，001-999）
- CCCC：居民顺序号（4位数字，0001-9999）

**示例**：`001-002-0001`（表示001街道办、002村委会的第1位居民）

**生成逻辑**：
- 选择街道办和村委会后，系统自动查询该村委会下现有居民的最大序号
- 新序号 = 最大序号 + 1
- 生成完整的用户编号

## 三、对各补贴身份的具体影响

### 3.1 失地居民（LandLossResident）

#### 当前字段结构
```java
- id
- subsidyPersonId (关联基础人员信息)
- landRequisitionTime (征地时间)
- compensationCompleteTime (完成补偿时间)
- recognitionTime (认定为失地居民时间)
- isUnder16 (是否16周岁以下)
- subsidyAmount (补贴金额)
- distributionStatus (发放状态)
- status (状态)
```

#### 本次修改内容

**删除字段**：
- ✅ `is_under_16` - 是否16周岁以下
- ✅ `subsidy_amount` - 补贴金额
- ✅ `distribution_status` - 发放状态
- ✅ `status` - 状态

**保留字段**：
- ✅ `land_requisition_time` - 征地时间
- ✅ `compensation_complete_time` - 完成补偿时间
- ✅ `recognition_time` - 认定为失地居民时间

**业务影响**：
- 补贴金额和发放状态改由"补贴发放管理"模块统一管理
- 状态管理简化，只保留基础人员信息的状态
- 去除16周岁以下的特殊标识
