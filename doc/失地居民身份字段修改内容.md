### 四、关键业务逻辑

#### 🔢 **用户编号生成规则**
```
格式：AAA-BBB-CCCC
- AAA：街道办编码（3位数字，如：001）
- BBB：村委会编码（3位数字，如：002）
- CCCC：居民顺序号（4位数字，从0001开始）

示例：001-002-0001（表示001街道办、002村委会的第1位居民）
```

#### 🔗 **级联选择逻辑**
1. 选择街道办 → 村委会下拉框动态加载该街道办下的村委会
2. 街道办和村委会确定后 → 自动生成用户编号

#### 👤 **是否健在联动逻辑**
- 选择"是" → 隐藏死亡时间字段
- 选择"否" → 显示死亡时间字段，且为必填项
