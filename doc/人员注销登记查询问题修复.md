# 人员注销登记列表查询问题修复

## 🐛 问题描述

**现象**: 在`shebao_person_cancel`表中新增了注销登记记录,但在前端页面的列表中显示"暂无数据",查询不到任何记录。

**原因**: 
1. **主要原因**: Controller返回的数据格式不正确
   - Controller直接返回MyBatis Plus的`Page`对象
   - 前端期望的是若依框架的`{rows: [], total: 0}`格式
   - 导致前端无法正确解析数据

2. **次要原因**: 在`PersonCancelServiceImpl.insertPersonCancel()`方法中插入记录时,**忘记设置`del_flag`字段**
   - 导致新插入的记录`del_flag`值为`NULL`
   - 查询SQL使用了`WHERE del_flag = '0'`条件过滤
   - `del_flag`为`NULL`的记录被排除

## ✅ 解决方案

### 1. 修复Controller返回格式 ⭐ (主要修复)

**文件**: `langfang-shebao/src/main/java/com/ruoyi/shebao/controller/PersonCancelController.java`

**修改位置**: `list()`方法第39-44行

**修改内容**:
```java
// 修改前 - 直接返回Page对象
@GetMapping("/list")
public AjaxResult list(PersonCancelListReq req)
{
    Page<PersonCancelListResp> page = personCancelService.selectPersonCancelList(req);
    return AjaxResult.success(page);  // ❌ 错误：返回格式不对
}

// 修改后 - 转换为若依框架格式
@GetMapping("/list")
public AjaxResult list(PersonCancelListReq req)
{
    Page<PersonCancelListResp> page = personCancelService.selectPersonCancelList(req);
    // 转换为若依框架的TableDataInfo格式：{rows: [], total: 0}
    return AjaxResult.success()
            .put("rows", page.getRecords())   // ✅ 正确：使用rows字段
            .put("total", page.getTotal());   // ✅ 正确：使用total字段
}
```

**说明**: 
- MyBatis Plus的`Page`对象使用`records`和`total`字段
- 若依前端期望的是`rows`和`total`字段
- 必须手动转换格式

### 3. 修复已有数据 (可选,如果del_flag为NULL)

**文件**: `sql/fix_person_cancel_del_flag.sql`

**执行SQL**:
```sql
-- 修复 shebao_person_cancel 表中 del_flag 为 NULL 的记录
UPDATE shebao_person_cancel 
SET del_flag = '0' 
WHERE del_flag IS NULL;
```

**说明**: 如果`del_flag`已经是`'0'`则无需执行此步骤。

## 🎓 经验教训

### 问题根源
1. **返回格式不匹配** ⭐ (主要问题):
   - MyBatis Plus的`Page`对象使用`records`字段存储数据列表
   - 若依前端框架期望`rows`字段
   - 直接返回`Page`对象会导致前端无法正确解析数据
   
2. **代码审查疏漏** (次要问题): 
   - 在Service实现类中插入实体时,忘记设置逻辑删除标志字段

3. **字段默认值依赖**: 
   - 虽然数据库表定义了DEFAULT值,但MyBatis Plus在INSERT时如果Java对象字段为null,可能不会使用数据库默认值

### 最佳实践

1. **Controller返回分页数据的正确写法**:
```java
@GetMapping("/list")
public AjaxResult list(XxxListReq req) {
    Page<XxxListResp> page = xxxService.selectXxxList(req);
    
    // ✅ 正确：转换为若依框架格式
    return AjaxResult.success()
            .put("rows", page.getRecords())
            .put("total", page.getTotal());
    
    // ❌ 错误：直接返回Page对象
    // return AjaxResult.success(page);
}
```

2. **新增实体时必须设置的字段**:
   - `delFlag` - 删除标志(默认'0')
   - `createBy` - 创建人
   - `createTime` - 创建时间
   - `status` - 状态字段(如果有)

3. **代码模板**:
```java
@Override
@Transactional(rollbackFor = Exception.class)
public int insertXxx(XxxFormDto formDto) {
    // 数据校验
    // ...
    
    Xxx entity = new Xxx();
    entity.setXxx(...);
    entity.setDelFlag("0");        // ✅ 必须设置
    entity.setStatus("0");         // ✅ 如果有状态字段
    entity.setCreateBy(SecurityUtils.getUsername());
    entity.setCreateTime(LocalDateTime.now());
    return xxxMapper.insert(entity);
}
```

4. **单元测试**: 应该为Service的insert方法编写单元测试,验证插入后能够正确查询到记录
