## 一、重构背景分析

### 1.1 系统现状

基于 Sequential Thinking 分析,当前系统存在以下特征:

**技术栈**:
- 框架: 若依(RuoYi) v3.9.0 (Spring Boot 3.5.4 + Vue 2.x)
- 数据库: MySQL 8.0 + MyBatis-Plus 3.5.6
- 前端: Vue 2.x + Element UI
- 架构: 前后端分离

**业务现状**:
- 支持4种补贴类型(失地、被征地、拆迁、村干部)
- 简单的2级审核流程
- 基础的角色权限控制

### 1.2 重构驱动因素

根据需求分析,重构主要由以下因素驱动:

1. **业务扩展需求**: 新增教师补贴,支持5种补贴类型
2. **流程复杂化**: 从2级审核升级为多级审批流程(2级、3级、4级)
3. **角色细化**: 从简单角色扩展为6个独立角色,明确职责边界
4. **审计合规**: 完整的审批追溯和操作日志记录

### 1.3 核心挑战识别

通过 Sequential Thinking 的分析阶段,识别出以下核心挑战:

| 挑战领域 | 复杂度 | 关键点 |
|---------|-------|--------|
| **审批流程引擎** | 高 | 状态机设计、多级审批、逐级退回 |
| **权限控制** | 中 | 6个角色、菜单权限、按钮权限 |
| **数据库设计** | 中 | 新增5表、修改2表、数据迁移 |
| **文件处理** | 高 | 批量导入、ZIP解压、文件管理 |
| **前端重构** | 中 | 菜单重构、路由权限、组件封装 |

## 二、技术架构决策

### 2.1 审批流程设计

**设计选择**: 基于状态机的简化实现(方案A)

**理由**:
- ✅ 快速实现,易于理解和维护
- ✅ 符合当前系统规模和复杂度
- ✅ 便于后续扩展和调整
- ❌ 不引入工作流引擎(避免过度设计)

**核心组件**:
```
ApprovalStatus (枚举)
  ├── DRAFT (草稿)
  ├── PENDING_REVIEW (待复核)
  ├── PENDING_APPROVE (待审批)
  ├── PENDING_FINANCE (待财务)
  ├── DISTRIBUTED (已发放)
  └── REJECTED (已驳回)

ApprovalStateMachine (状态机)
  └── canTransition() (状态流转验证)

ApprovalService (审批服务)
  ├── submitForReview() (提交审核)
  ├── review() (复核)
  ├── approve() (审批)
  ├── reject() (驳回)
  └── cancel() (撤销)
```

### 2.2 数据库设计策略

**设计选择**: 最小化变更(方案A)

**新增表**(5张):
1. `approval_log` - 审批流程记录表
2. `distribution_batch` - 发放批次表
3. `approval_role_config` - 审批角色配置表
4. `finance_account` - 财务账户表
5. `benefit_determination` - 待遇核定表

**修改表**(2张):
1. `shebao_subsidy_distribution` - 添加审批字段
2. `shebao_subsidy_person` - 添加认证和暂停状态字段

### 2.3 权限控制架构

**三层权限控制**:
```
1. 菜单权限 (Menu Level)
   - 基于角色的动态菜单加载
   - sys_menu + sys_role_menu

2. 接口权限 (API Level)
   - Spring Security @PreAuthorize
   - 方法级权限控制

3. 按钮权限 (Button Level)
   - Vue指令 v-if + hasPermi()
   - 前端按钮显示/隐藏
```

## 四、关键技术实现

### 4.1 审批流程实现要点

**状态流转验证**:
```java
public class ApprovalStateMachine {
    public boolean canTransition(ApprovalStatus current,
                                 ApprovalStatus target,
                                 String operation) {
        // 草稿 → 待复核
        if (current == DRAFT && target == PENDING_REVIEW
            && "submit".equals(operation)) {
            return true;
        }
        // 待复核 → 待审批
        if (current == PENDING_REVIEW && target == PENDING_APPROVE
            && "review".equals(operation)) {
            return true;
        }
        // 待复核 → 草稿 (驳回)
        if (current == PENDING_REVIEW && target == DRAFT
            && "reject".equals(operation)) {
            return true;
        }
        // ... 其他状态流转规则
        return false;
    }
}
```

**审批日志记录**:
```java
@Transactional
public void submitForReview(Long businessId, String businessType) {
    // 1. 验证状态
    validateStatus(business);

    // 2. 更新业务状态
    updateStatus(businessId, businessType, ApprovalStatus.PENDING_REVIEW);

    // 3. 记录审批日志
    ApprovalLog log = new ApprovalLog();
    log.setBusinessType(businessType);
    log.setBusinessId(businessId);
    log.setOperationType("submit");
    log.setOperatorId(SecurityUtils.getUserId());
    approvalLogMapper.insert(log);
}
```

### 4.2 权限控制实现要点

**后端权限控制**:
```java
@RestController
@RequestMapping("/approval")
public class ApprovalController {

    @PreAuthorize("@ss.hasPermi('shebao:approval:review')")
    @PostMapping("/review")
    public AjaxResult review(@RequestBody Map<String, Object> params) {
        // 复核逻辑
        approvalService.review(businessId, businessType, approved, remark);
        return toAjax(true);
    }

    @PreAuthorize("@ss.hasPermi('shebao:approval:approve')")
    @PostMapping("/approve")
    public AjaxResult approve(@RequestBody Map<String, Object> params) {
        // 审批逻辑
        approvalService.approve(businessId, businessType, approved, remark);
        return toAjax(true);
    }
}
```

**前端权限控制**:
```vue
<template>
  <div>
    <!-- 根据状态和权限显示按钮 -->
    <el-button
      v-if="record.approvalStatus === 'pending_review'
            && hasPermi('shebao:approval:review')"
      type="primary"
      @click="handleReview(record)">
      复核
    </el-button>

    <el-button
      v-if="record.approvalStatus === 'pending_approve'
            && hasPermi('shebao:approval:approve')"
      type="success"
      @click="handleApprove(record)">
      审批
    </el-button>
  </div>
</template>
```

### 4.3 批量导入实现要点

```java
@Service
public class BenefitDeterminationService {

    @Transactional
    public AjaxResult batchDetermination(MultipartFile file) {
        try {
            // 1. 读取Excel文件
            List<BenefitDeterminationExcel> excelList =
                ExcelUtils.importExcel(file, BenefitDeterminationExcel.class);

            // 2. 数据校验
            validateExcelList(excelList);

            // 3. 批量插入
            List<BenefitDetermination> benefitList = new ArrayList<>();
            for (BenefitDeterminationExcel excel : excelList) {
                BenefitDetermination benefit = convertToEntity(excel);
                benefitList.add(benefit);
            }
            benefitDeterminationMapper.insertBatch(benefitList);

            // 4. 提交审核
            for (BenefitDetermination benefit : benefitList) {
                approvalService.submitForReview(
                    benefit.getId(), "benefit_determination");
            }

            return AjaxResult.success("成功导入" + benefitList.size() + "条数据");
        } catch (Exception e) {
            throw new ServiceException("批量导入失败: " + e.getMessage());
        }
    }
}
```
