```java
package com.ruoyi.shebao.approval.enums;

/**
 * 审批状态枚举
 */
public enum ApprovalStatus {
    DRAFT("draft", "草稿"),
    PENDING_REVIEW("pending_review", "待复核"),
    PENDING_APPROVE("pending_approve", "待审批"),
    PENDING_FINANCE("pending_finance", "待财务"),
    DISTRIBUTED("distributed", "已发放"),
    REJECTED("rejected", "已驳回");

    private final String code;
    private final String name;

    ApprovalStatus(String code, String name) {
        this.code = code;
        this.name = name;
    }

    public String getCode() { return code; }
    public String getName() { return name; }
}
```

#### Step 3: 创建状态机

```java
// 3. 创建 ApprovalStateMachine.java
package com.ruoyi.shebao.approval.statemachine;

import com.ruoyi.shebao.approval.enums.ApprovalStatus;
import org.springframework.stereotype.Component;

/**
 * 审批状态机
 */
@Component
public class ApprovalStateMachine {

    /**
     * 验证状态流转是否合法
     */
    public boolean canTransition(ApprovalStatus current,
                                 ApprovalStatus target,
                                 String operation) {
        // 草稿 → 待复核 (提交)
        if (current == ApprovalStatus.DRAFT
            && target == ApprovalStatus.PENDING_REVIEW
            && "submit".equals(operation)) {
            return true;
        }

        // 待复核 → 待审批 (复核通过)
        if (current == ApprovalStatus.PENDING_REVIEW
            && target == ApprovalStatus.PENDING_APPROVE
            && "review".equals(operation)) {
            return true;
        }

        // 待复核 → 草稿 (复核驳回)
        if (current == ApprovalStatus.PENDING_REVIEW
            && target == ApprovalStatus.DRAFT
            && "reject".equals(operation)) {
            return true;
        }

        // 待审批 → 待财务 (审批通过)
        if (current == ApprovalStatus.PENDING_APPROVE
            && target == ApprovalStatus.PENDING_FINANCE
            && "approve".equals(operation)) {
            return true;
        }

        // 待审批 → 待复核 (审批驳回)
        if (current == ApprovalStatus.PENDING_APPROVE
            && target == ApprovalStatus.PENDING_REVIEW
            && "reject".equals(operation)) {
            return true;
        }

        // 待财务 → 已发放 (财务发放)
        if (current == ApprovalStatus.PENDING_FINANCE
            && target == ApprovalStatus.DISTRIBUTED
            && "distribute".equals(operation)) {
            return true;
        }

        // 待财务 → 待审批 (财务驳回)
        if (current == ApprovalStatus.PENDING_FINANCE
            && target == ApprovalStatus.PENDING_APPROVE
            && "reject".equals(operation)) {
            return true;
        }

        return false;
    }

    /**
     * 获取下一个状态
     */
    public ApprovalStatus getNextStatus(ApprovalStatus current,
                                        String operation,
                                        boolean approved) {
        if ("submit".equals(operation)) {
            return ApprovalStatus.PENDING_REVIEW;
        }

        if ("review".equals(operation)) {
            return approved ? ApprovalStatus.PENDING_APPROVE : ApprovalStatus.DRAFT;
        }

        if ("approve".equals(operation)) {
            return approved ? ApprovalStatus.PENDING_FINANCE : ApprovalStatus.PENDING_REVIEW;
        }

        if ("distribute".equals(operation)) {
            return ApprovalStatus.DISTRIBUTED;
        }

        return current;
    }
}
```

## 📝 开发规范

### 代码规范

1. **命名规范**:
   - 类名: 大驼峰 `ApprovalService`
   - 方法名: 小驼峰 `submitForReview`
   - 常量: 全大写下划线 `MAX_RETRY_COUNT`
   - 包名: 全小写 `com.ruoyi.shebao.approval`

2. **注释规范**:
   ```java
   /**
    * 提交审核
    *
    * @param businessId 业务ID
    * @param businessType 业务类型
    * @return 操作结果
    * @throws ServiceException 业务异常
    */
   @Transactional
   public AjaxResult submitForReview(Long businessId, String businessType) {
       // 方法实现
   }
   ```

3. **日志规范**:
   ```java
   log.info("提交审核: businessType={}, businessId={}", businessType, businessId);
   log.error("审核失败: {}", e.getMessage(), e);
   ```

### Git提交规范

**提交信息格式**:
```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type类型**:
- `feat`: 新功能
- `fix`: 修复Bug
- `docs`: 文档更新
- `style`: 代码格式(不影响代码运行)
- `refactor`: 重构
- `test`: 测试用例
- `chore`: 构建过程或辅助工具的变动

**示例**:

## ❓ 常见问题

### Q1: 数据库脚本执行失败?

**A**: 检查以下几点:
1. MySQL版本是否为8.0+
2. 字符集是否为utf8mb4
3. 表是否已存在(删除后重新创建)
4. 外键约束是否冲突

### Q2: 后端服务启动失败?

**A**: 检查:
1. JDK版本是否为17+
2. Maven依赖是否下载完整
3. 数据库连接配置是否正确
4. Redis是否启动(如果使用)

### Q3: 前端页面显示404?

**A**: 检查:
1. 路由配置是否正确
2. 菜单权限是否分配
3. npm是否安装完整
4. 前端服务是否启动

### Q4: 审批流程测试失败?

**A**: 检查:
1. 用户角色是否正确
2. 审批权限是否分配
3. 状态流转是否符合规则
4. 审批日志是否记录
