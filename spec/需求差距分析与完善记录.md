## 二、已发现并已完善的差距

### 1. 补贴类型枚举缺少「教龄补助」（类型 5）——已完善

| 项目 | 说明 |
|------|------|
| **需求** | 用户需求说明书与概要设计均规定 5 种补贴类型，第 5 种为「教龄补助」（原“教师补贴”）。数据字典 `subsidy_type` 含 1～5。 |
| **原状** | `SubsidyTypeEnum` 仅包含 1～4（失地、被征地、拆迁、村干部），无类型 5。 |
| **完善** | ① 在 `SubsidyTypeEnum` 中新增 `TEACHER("5", "教龄补助", "shebao_teacher_subsidy")`。② `SubsidyDistributionServiceImpl`：注入 `TeacherSubsidyMapper`，在可用补贴列表中加入教龄补助查询；在 `validateSubsidyRecord`、`updateSubsidyRecordStatus`、`getLatestSubsidyAmount` 的 switch 中增加 `TEACHER` 分支。③ `BenefitDeterminationServiceImpl`：解析补贴类型时增加 `"teacher_subsidy" -> SubsidyTypeEnum.TEACHER`。④ `SubsidyDistribution` 实体：Excel 与注释中补充「5=教龄补助」。 |
| **验证** | 新增 JUnit 测试 `SubsidyTypeEnumTest.shouldHaveFiveTypesIncludingTeacher()`，断言类型 5 存在且名称为「教龄补助」。 |

## 三、待后续核对的差距（建议逐项完善）

以下项需结合现有代码与数据库进一步确认，并在实现或迭代时完善。

1. **发放记录与审批状态**  
   需求：发放流程有多级审批状态（草稿、待复核、待审批、待财务、已发放、已驳回）。  
   核对：`SubsidyDistribution`、`DistributionBatch` 等是否使用与概要设计一致的 `approval_status` 及状态流转。

2. **财务账户与 5 个补贴类型**  
   需求：5 种补贴对应 5 个财务账号。  
   核对：`finance_account` 表及 `FinanceAccount` 是否按补贴类型 1～5 配置并可区分教龄补助。

3. **教龄补助业务逻辑**  
   需求：教龄补助无登记核定，发放明细由他部门提供，系统做导入、认证、统计查询。  
   核对：教龄补助是否仅做导入与查询，不参与「登记→核定→发放」完整流程；菜单与权限是否单独配置。

4. **被征地参保补贴独立模块**  
   需求：独立流程（登记→申领→支付计划→综合查询），接口命名空间如 `/shebao/landAcq/*`。  
   核对：是否存在对应 Controller/Service 及路由，是否与现有「被征地居民」区分。

5. **角色与菜单**  
   需求：6 类角色（经办、复核、审批、财务、统计管理员、系统管理员）及不同菜单。  
   核对：系统角色、菜单及权限标识是否与概要设计 6.1～6.4 节一致。

6. **待遇核定与发放**  
   需求：到龄年月、享受开始年月、补发规则、银行信息、档案上传等。  
   核对：`BenefitDetermination` 及相关接口是否覆盖上述字段与流程。

## 四、测试与运行说明

### 运行 JUnit 测试

在项目根目录执行（需能解析 Maven 依赖，如内网 Nexus 或阿里云镜像）：

```bash
# 仅运行 langfang-shebao 模块测试
mvn -pl langfang-shebao test

# 运行全部模块测试
mvn test
```

当前已编写的测试类：

- `com.ruoyi.shebao.service.impl.SubsidyCalculationServiceImplTest`：补贴年限、年龄、金额、年平均工资
- `com.ruoyi.shebao.enums.SubsidyTypeEnumTest`：补贴类型编码与表名、5 种类型含教龄补助

### 运行当前项目

- **后端**：`mvn spring-boot:run -pl ruoyi-admin`（需配置数据库等，见 `spec/快速启动指南.md`）
- **前端**：`cd ruoyi-ui && npm install && npm run dev`
