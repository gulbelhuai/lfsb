# 预到龄发放通知使用说明

**文档版本**: v2.0  
**创建日期**: 2026-03-15  
**适用对象**: 经办人、系统管理员、测试人员

---

## 一、功能简介

“预到龄发放通知”用于生成按月管理的预到龄通知批次，并将该批次人员直接衔接到“待遇核定”“待遇核定复核”“支付计划”。

当前系统中的该功能已支持：

- 按 `通知月份` 生成通知批次
- 按月份、批次号查询历史批次
- 查看批次明细
- 导出 Excel 清单
- 在待遇核定页面按 `通知批次号` 录入或批量导入信息
- 上传 `用户ID.zip` 并在线预览图片
- 在待遇核定复核页面按批次批量审核通过
- 审核通过后按批次生成支付计划

---

## 二、适用角色与权限

### 2.1 页面访问权限

- `shebao:benefit:notice:list`

### 2.2 核定和复核权限

- 通知生成：`shebao:benefit:notice:generate`
- 核定导入/材料上传：`shebao:benefit:determination:import`
- 核定提交：`shebao:benefit:determination:submit`
- 复核通过/驳回：`shebao:benefit:review:approve`

### 2.3 使用建议

- 经办人至少同时具备列表和生成两项权限
- 若只有列表权限，则可查询但不一定能成功调用生成接口
- 若环境未配置按钮级权限，页面上可能仍会显示“生成通知”按钮

## 三、页面入口

系统菜单路径：

`待遇核定管理` -> `预到龄发放通知`

前端页面位置：

`ruoyi-ui/src/views/shebao/benefit/notice/index.vue`

## 四、页面说明

### 4.1 顶部操作区

页面顶部包含以下控件：

| 控件 | 说明 |
|------|------|
| 通知月份 | 必填，表示当前通知业务月份 |
| 年龄阈值 | 默认 60 |
| 生成通知 | 幂等生成当月通知批次 |
| 批次号 | 查询指定批次 |
| 导出清单 | 导出当前批次或当前筛选结果 |

### 4.2 查询结果区

列表展示当前筛选条件下的通知批次。常见字段含义如下：

| 字段 | 说明 |
|------|------|
| 批次号 | 当次通知业务唯一编号 |
| 通知月份 | 当前业务月份 |
| 到龄月份 | 通知月份 + 3个月 |
| 总人数 | 批次内总人数 |
| 待复核人数 | 当前批次中待复核的核定记录数 |
| 已通过人数 | 当前批次中已通过的核定记录数 |

## 五、操作步骤

### 5.1 生成通知批次

1. 在页面选择通知月份。
2. 保持或调整“年龄阈值”。
3. 点击“生成通知”。
4. 在确认框中点击确认。
5. 系统生成 `通知批次 + 通知明细 + 待遇核定草稿`。

### 5.2 查询批次和查看明细

1. 在页面选择通知月份或输入批次号。
2. 点击搜索。
3. 在批次列表中点击“查看明细”。
4. 在明细弹窗中查看人员、核定状态、材料状态。

### 5.3 进入待遇核定

1. 在通知页面点击“去核定”。
2. 系统自动带入 `通知批次号` 到待遇核定页面。
3. 对每名人员录入身份证号、银行卡号、账户名、补贴标准等信息。
4. 上传 `用户ID.zip`，系统自动解压并展示图片预览。
5. 保存后点击“提交复核”。

### 5.4 批量导入

1. 在待遇核定页面点击“批量导入”。
2. 下载模板并按要求填写。
3. 选择 Excel 文件。
4. 可同时选择多份 `用户ID.zip` 材料包。
5. 点击开始导入，系统将自动匹配批次和用户，并提交待复核。

### 5.5 复核与进入支付计划

1. 在“待遇核定复核”页面按 `通知批次号` 查询。
2. 抽查单条详情、材料图片和原 ZIP。
3. 逐条通过/驳回，或多选后执行“批量通过”。
4. 审核通过后，可在支付计划接口按批次生成支付计划。

## 六、当前筛选口径

当前系统生成批次时会从被补贴人主表中筛选符合条件的人员，主要规则为：

- 人员未删除
- 人员健在
- 审批状态为 `approved`
- `birthday + 60年` 所在月份 = `通知月份 + 3个月`

---

## 七、注意事项

### 7.1 关于通知月份

- 通知月份建议明确指定
- 系统按“通知月份 + 3个月”确定实际到龄月份

### 7.2 关于 ZIP 命名

- 单个材料包必须命名为 `用户ID.zip`
- ZIP 内建议放身份证、银行卡等图片

### 7.3 关于导入

- Excel 行数据必须与当前批次人员一一对应
- 若批次号、用户 ID、身份证号或银行卡号错误，系统会返回错误

### 7.4 关于复核

- 只有 `pending_review` 状态的数据可在复核页处理
- 只有 `approved` 状态的数据可以进入支付计划

## 八、常见使用场景

### 8.1 经办人月初生成通知

适用方式：

- 输入目标月份
- 生成通知批次
- 导出 Excel 给线下回填

### 8.2 经办人批量回填核定信息

建议方式：

- 批量导入 Excel
- 批量上传 `用户ID.zip`
- 自动提交待复核

### 8.3 复核人按批次抽查

建议方式：

- 输入批次号
- 抽查图片材料
- 批量审核通过

## 九、异常与处理建议

### 9.1 页面无数据

排查顺序：

1. 是否选择了正确的通知月份
2. 样本人员是否满足 `approved`、`is_alive='1'`、`del_flag='0'`
3. 样本生日是否落入当前实现的窗口

### 9.2 导入失败

优先检查：

1. Excel 中的批次号和用户 ID 是否准确
2. ZIP 是否命名为 `用户ID.zip`
3. ZIP 中是否存在图片文件
当前版本中“生成通知”仅返回成功消息，没有实际落库。

### 9.3 看得到按钮但点击报无权限

这是当前实现可能存在的权限表现差异。  
应检查：

- 账号是否拥有 `shebao:benefit:notice:generate`
- 页面是否已接入按钮级权限控制

### 9.4 修改年龄阈值后结果未变化

这是当前实现中的已知限制。  
目前不应把“年龄阈值”当作已完整生效的业务参数。

## 十、测试使用建议

为了保证测试结果稳定，建议按以下方式使用：

1. 指定明确的 `noticeMonth`
2. 准备可命中与排除型样本各一组
3. 先验证列表，再验证生成
4. 报告中分开记录“接口返回成功”和“业务结果已落地”

---

## 十一、总结

当前版本的“预到龄发放通知”适合作为：

- 预到龄候选人员查询入口
- 业务流程前置校验页面
- 后续完整通知功能的占位模块

当前版本不适合作为：

- 正式通知台账
- 通知状态管理工具
- 可审计的通知批次管理功能
